# 분석 문서: /today — 오늘의 매거진

작성일: 2026-06-17 | 분석 기반: 코드 (git main)

---

## 1. 페이지 위치·역할

- **URL**: `/today` (쿼리: `?tab=executive|staff|cs`, `?date=YYYY-MM-DD`)
- **사이드바 메뉴**: 오늘의 매거진 (IcSpark 아이콘)
- **권한**: 로그인 사용자 전체
- **기본값**: tab=executive, date=KST 오늘

AI가 매일 06:00 KST에 무신사 수집 데이터를 분석해 자동 생성한 일일 브리핑을 부서별로 제공하는 허브 페이지. 경영진·기획/영업·CS 세 부서 맞춤 요약을 각각 독립 탭으로 제공.

---

## 2. 화면 구성 (코드 기반)

### 상단: 날짜 네비게이션 + 탭 바
- `BriefingDateNav`: 날짜 선택 (최근 60일치 브리핑 날짜만 선택 가능)
- 탭 3개: **경영진** / **기획/영업** / **CS**
  - 탭 버튼에 해당 날짜 헤드라인 미리보기(11px)가 표시됨

### 경영진 탭 (executive)
- `BriefingHeadline`: 한 줄 핵심 요약 + 상세 브리프 목록
- 카드 4개 (해당 날짜 데이터 있을 때만 표시): 경쟁 브랜드 동향 / 외부 뉴스 / 자사 랭킹 / 이상 탐지
- `BriefingKpiRow`: 자사 브랜드 현황·이상탐지 수·경쟁 상위 5개 브랜드 KPI 행
- `NewsPickList`: AI 선별 뉴스 (있을 때만)
- `BriefingInsight`: 인사이트 섹션 (있을 때만)

### 기획/영업 탭 (staff)
- 헤드라인 + 카드 8개: 자사 랭킹 / 프로모션 / 이상 탐지 / 고객 리뷰 / 경쟁 브랜드 동향 / 트렌드 / DART 공시 / 외부 뉴스
- KPI 행 + 인사이트

### CS 탭 (cs)
- 헤드라인 + 카드 4개: 오늘의 리뷰 / 저점 패턴 / 고점 패턴 / 문제 상품
- 인사이트 (있을 때만)

### 모바일
- `MobileTodayView` 컴포넌트로 분기 (≤767px)

---

## 3. 데이터 흐름

- 테이블: `daily_briefings` (audience별 row, briefing_date 기준)
- `fetchAvailableBriefingDates()`: 최근 60일 날짜 목록 (날짜 선택 UI용)
- `fetchAllBriefings(date)`: 해당 날짜의 executive/staff/cs 3건 조회
- `fetchBriefingKpiData(date)`: KPI 지표 별도 쿼리
- 실시간 갱신 없음 (페이지 로드 + 날짜 변경 시 fetch)

---

## 4. 사용자 인터랙션

- **탭 전환**: URL `?tab=` 업데이트 (스크롤 위치 유지)
- **날짜 변경**: URL `?date=YYYY-MM-DD` 업데이트 → 해당 날짜 브리핑 로드
- **카드 클릭**: 각 카드는 관련 페이지로 이동 (예: 자사 랭킹 → `/ranking`)
- **미래 날짜 선택 시**: FutureEmptyState 표시 (조회 불가 안내)

---

## 5. 빈 상태·에러 처리

- 당일 브리핑 미생성: "YYYY-MM-DD 브리핑이 아직 생성되지 않았습니다. 매일 06:00 자동 생성됩니다."
- 미래 날짜: "미래 날짜의 브리핑은 조회할 수 없습니다."
- 카드: 해당 카드 데이터 없으면 카드 자체를 숨김

---

## 6. 관련 기능

- 랭킹 카드 → `/ranking`
- 브랜드 랭킹 카드 → `/brand-ranking`
- 이상 탐지 카드 → `/anomaly`
- 리뷰 카드 → `/reviews`
- 프로모션 카드 → `/promo`
- 매거진/뉴스 카드 → `/magazine`
- 회사/DART 카드 → `/company`

---

## 7. 사용자가 자주 헷갈리는 부분

- "오늘 브리핑이 왜 없나요?" → 06:00 자동 생성이므로 새벽에는 아직 없을 수 있음
- "탭을 바꿔도 날짜가 유지되나요?" → 유지됨 (URL 파라미터)
- "어제 브리핑을 다시 볼 수 있나요?" → 최근 60일치 조회 가능
- 탭별 카드 구성이 다름 — 직무별로 맞춤 설계됨
