# 분석 문서: /ranking — 상품 랭킹

작성일: 2026-06-17 | 분석 기반: 코드 (git main)

---

## 1. 페이지 위치·역할

- **URL**: `/ranking`
- **사이드바 메뉴**: 상품 랭킹 (IcRanking 아이콘)
- **권한**: 로그인 사용자 전체
- **기본값**: 오늘 / 전체 카테고리 / 전체 성별 / 전체 연령 / 순위 오름차순

무신사에서 매일 수집한 상품 랭킹 스냅샷을 필터링·정렬하는 핵심 조회 페이지.
자사 상품의 위치, 경쟁 상품 현황, 가격·할인 정보를 한 화면에서 확인 가능.

---

## 2. 화면 구성 (코드 기반)

### 좌측: 필터 레일
- **기간**: 오늘 / 7일 / 30일 / 90일 / 직접 입력
- **카테고리**: 전체·상의·아우터·바지·가방·스포츠/레저·속옷/홈웨어·원피스/스커트·소품·디지털/라이프·신발·뷰티·키즈
- **성별**: 전체·남성·여성
- **연령대**: 전체·20세 미만·20~25세·25~30세·30~35세·35~40세·40세 이상
- **가격대**: 슬라이더 + 직접 입력 (0~50만원+)
- **회사**: 검색 추가 (다중 선택)
- **브랜드**: 검색 추가 (다중 선택)
- **옵션**: 자사 상품만 / 변동 항목만
- **저장된 필터**: SavedFiltersDropdown — 자주 쓰는 필터 세트 저장/불러오기

### 우측: 결과 테이블
- 적용 필터 칩 + 정렬 버튼 (순위/변동/리뷰점수/리뷰수/상품명)
- 컬럼: 순위 / 변동(▲/▼/NEW) / 이미지 / 상품명(자사=하이라이트) / 브랜드·회사 / 카테고리 / 성별·연령 / 판매가격·소비자가 / 할인율 / 리뷰점수 / 리뷰수
- 50개씩 페이지네이션, 최대 300개 fetch
- 날짜 선택 시 날짜 컬럼 추가

### 우상단 버튼
- CSV 다운로드 (현재 필터 결과 전체)
- 메모 (현재 필터 조합에 메모 추가)

---

## 3. 데이터 흐름

- 서버 쿼리: 카테고리·성별·연령·기간 변경 시 `fetchLatestRanking()` 재호출 (최대 300건)
- 클라이언트 필터: 회사·브랜드·가격·자사/변동 — 메모리에서 즉시 반영
- 필터 상태 localStorage 자동 저장 (`uttu-ranking-filters`, 사용자ID 바인딩)
- 다른 계정 로그인 시 저장된 필터 자동 초기화

---

## 4. 사용자 인터랙션

- **행 클릭** → `/product?no=무신사번호` 상품 상세
- **이미지 클릭** → 라이트박스 (전체 화면 이미지)
- **필터 칩 ✕** → 해당 필터만 즉시 해제
- **[초기화]** → 모든 필터 기본값으로
- **저장된 필터** → 자주 쓰는 필터 조합 이름 저장·불러오기

---

## 5. 빈 상태·에러 처리

- 수집 데이터 없음: "수집된 랭킹 데이터가 없습니다."
- 필터 조건 불일치: "조건에 맞는 상품이 없습니다. 필터를 완화해 보세요."
- 로드 중: 스켈레톤 10행 표시

---

## 6. 관련 기능

- 상품 클릭 → `/product?no=XXX` (상품 상세, 가격 추이 등)
- 홈 → 오늘의 상품랭킹 TOP10 위젯
- 오늘의 매거진 → 자사 랭킹 카드

---

## 7. 사용자가 자주 헷갈리는 부분

- "회사 필터와 브랜드 필터 차이?" → 회사=법인명 (예: 주식회사 에이비씨), 브랜드=상품 브랜드 (예: COVERNAT)
- "필터 적용해도 결과가 안 바뀌는 경우?" → 회사·브랜드·가격은 클라이언트 필터이므로 서버 재조회 없이 즉시 반영됨
- "기간을 7일로 바꾸면 같은 상품이 여러 행으로 나타남" → 날짜별 스냅샷이 각각 표시되어 날짜 컬럼이 추가됨
- "가격 50만+ 의미?" → 50만원 이상 상품이 모두 포함됨
