# 분석 문서: /brand-ranking — 브랜드 랭킹

작성일: 2026-06-17 | 분석 기반: 코드 (git main)

---

## 1. 페이지 위치·역할

- **URL**: `/brand-ranking`
- **사이드바 메뉴**: 브랜드 랭킹 (IcBrandRanking 아이콘)
- **권한**: 로그인 사용자 전체
- **기본값**: 오늘 / 전체 카테고리 / 전체 성별 / 전체 연령 / TOP100 수 기준 정렬

상품 랭킹 데이터를 브랜드 단위로 집계하여 보여주는 페이지.
TOP100 진입 상품 수, 평균 순위, 최고 순위, SKU 수 등을 한눈에 비교 가능.
기간 선택 시 이전 기간과의 변동폭(▲/▼)을 함께 표시함.

---

## 2. 화면 구성 (코드 기반)

### 좌측: 필터 레일
- **기간**: 오늘 / 7일 / 30일 / 90일 / 직접 입력 (시작~종료일)
  - 기간 선택 시 해당 기간 시작일과 비교 → 변동 컬럼 표시됨
- **카테고리**: 13개 카테고리
- **성별**: 전체·남성·여성
- **연령대**: 전체 + 6개 연령대
- **회사**: 검색 추가 (다중 선택)
- **옵션**: 자사 브랜드만

### 우측: 브랜드 테이블
- 정렬 기준 4가지: 순위 (TOP100 수) / 상품평균 (평균 순위) / 상품최고 (최고 순위) / 랭킹상품수 (SKU)
- 컬럼: 순위 / 브랜드명(자사=하이라이트) / 국가 / 회사 / TOP100 수(변동) / 평균 순위(변동) / 최고 순위 / SKU 수 / 평균 할인율 / 평균 가격·범위 / 평균 리뷰점수 / 리뷰수 합계
- 행 클릭 → `/brand?id=브랜드ID`

### 우상단 버튼
- CSV 다운로드
- 북마크(bookmark) [확인 필요 — 실제 기능 확인 필요]

## 3. 데이터 흐름

- `fetchBrandLeaderboard()`: 카테고리·성별·연령·기간 변경 시 서버 재쿼리
- `compareDate`: 기간 시작일로 계산 → 변동폭 산출
- 회사 필터: 클라이언트 사이드 적용
- localStorage 자동 저장 (`uttu-brand-ranking-filters`)

---

## 4. 사용자 인터랙션

- **행 클릭** → `/brand?id=브랜드ID` 브랜드 상세
- **정렬 버튼** → 해당 기준으로 즉시 재정렬
- **기간 변경** → 서버 재쿼리 + 변동 컬럼 자동 표시/숨김
- **[초기화]** → 모든 필터 기본값 복원

---

## 5. 빈 상태·에러 처리

- 로드 중: 로딩 표시
- 데이터 없음: 빈 테이블 (별도 메시지 없음 — [확인 필요])

---

## 6. 관련 기능

- 브랜드 행 클릭 → `/brand?id=XXX` (브랜드 상세 페이지)
- 홈 → 브랜드 랭킹 TOP10 위젯
- 오늘의 매거진 → 경쟁 브랜드 동향 카드

---

## 7. 사용자가 자주 헷갈리는 부분

- "TOP100 수와 SKU 수 차이?" → TOP100 수=TOP100 안에 진입한 상품 개수, SKU 수=해당 브랜드의 랭킹에 집계된 전체 상품 수
- "변동이 표시되지 않을 때?" → 기간을 '오늘'로 선택하면 비교 기준이 없어 변동 컬럼이 숨겨짐
- "평균 순위가 낮을수록 좋은 건가요?" → 순위는 숫자가 낮을수록 상위. 평균 순위 '3.5'는 평균적으로 3~4위권임을 의미함
